JavaScript那些事儿(一)引用类型和基本类型

发表于 2017-05-26   |   分类于 Javascript那些事儿   |  

ES变量包含两种不同数据类型的值：基本类型值和引用类型值。

定义：

  1. 基本数据类型：简单的数据段。Undefined、Null、Boolean、Number、String.
  2. 引用数据类型：由多个值构成的对象。除过上面的几种基本数据类型外，剩下的就是引用类型了，统称为Object类型。细分的话，有：Object 类型、Array 类型、Date 类型、RegExp 类型、Function 类型 等。

区别：

  1. 基本数据类型是按值访问的，因为可以操作保存在变量中的实际的值；
  2. 引用类型的值是按引用访问的，值是保存在内存中的对象，JavaScript不允许直接访问内存中的位置，即不能直接操作对象的内存空间所以在操作对象时实际上实在操作对象的引用而不是实际的对象。

动态属性：

  1. 基本数据类型不能添加动态属性值。

  2. 引用类型的值可以添加删除动态属性和方法。

复制变量值：

  1. 基本数据类型复制会在变量对象上创建一个新值，然后把该值放在新变量分配的位置上。

    此后num1和num2各自的操作不会影响另一方。

  2. 引用类型的复制 同样也会把存储在变量对象中的值复制一份到为新变量分配的空间中。不同的是：这个值的副本实际上是一个指针，这个指针指向存储在堆中的一个对象。复制结束以后，两个变量（指针）将引用同一个对象。互相影响。比如：

    在这里可能会有疑问,局部作用域的修改反映到全局作用域，不就是说明参数是按引用传递的吗？并不。如果在局部作用域中把一个新的对象赋值给obj重新修改name值，如果参数是按引用传递的，那么外部的person就会自动修改指向新的obj，但事实是即使修改了name值，原始的引用还是不变，name值还是不变。
